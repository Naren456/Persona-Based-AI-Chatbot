# Reflection

This assignment worked best when I treated the chatbot as a product problem, not only a prompt-writing exercise. A generic prompt such as "act like a helpful founder" would produce generic answers, so I first separated each persona by background, values, and likely teaching lens. Anshuman's prompt is more systems and fundamentals focused because his public background includes competitive programming, Facebook Messenger, and course design. Abhimanyu's prompt is more outcome and entrepreneurship focused because his public profile emphasizes building Scaler/InterviewBit, the education-to-employability gap, and practical risk-taking. Kshitij's prompt is more implementation and mentorship focused because public Scaler material connects him to engineering, teaching, Snapdeal, and learn-by-building.

The GIGO principle was the clearest lesson. The model can only imitate the shape of the instructions it receives. When the prompt contains shallow input, the answer becomes vague motivation. When the prompt contains constraints, examples, response length, a reasoning instruction, and clear boundaries, the output becomes more consistent and useful. The few-shot examples were especially important because they define not just content, but rhythm: how direct the answer should be, how much detail to include, and what kind of question should end the response.

On the product side, persona switching needed to reset the conversation because mixing old messages with a new system prompt would confuse the model and weaken persona consistency. The backend also keeps prompts server-side so the frontend never has to expose prompt logic or API keys. Error handling matters because a broken API call should feel like a recoverable service issue, not an app crash.

If I had more time, I would improve the research layer with more primary-source videos, public talks, and verified posts for each person. I would also add automated tests for the chat endpoint, deploy both services, add real screenshots, and collect sample conversations from people familiar with Scaler to tune authenticity. The biggest improvement would be a small evaluation checklist that scores each reply on persona fit, factual safety, actionability, and whether it respects the "simulation, not real person" boundary.
